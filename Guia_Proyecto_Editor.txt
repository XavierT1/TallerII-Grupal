GUÍA COMPLETA: EDITOR DE IMÁGENES UNIVERSITARIO (MODO AVANZADO)
================================================================================

1. DESCRIPCIÓN GENERAL
--------------------------------------------------------------------------------
Este proyecto es una aplicación de escritorio profesional desarrollada en Java
que permite la edición y procesamiento de imágenes digitales. Combina algoritmos
académicos de procesamiento de señales con una interfaz moderna y funcional.

2. FUNCIONALIDADES CLAVE
--------------------------------------------------------------------------------
- Procesamiento de Imágenes: Aplicación de filtros de color, efectos visuales 
  y matrices de convolución.
- Modo de Análisis: Vista en cuadrícula (grid) para comparar el efecto de un 
  filtro con diferentes intensidades (niveles N).
- Editor Avanzado (Modo Apilado): Permite aplicar múltiples filtros uno encima
  de otro, con un historial de cambios y función de deshacer (Undo).
- Herramientas Dinámicas: Sliders en tiempo real para ajustar transparencia, 
  máscaras de bits y tonalidades RGB.
- Comparativa Instantánea: Botón "Mantener para ver Original" para comparar
  rápidamente los cambios.
- Interfaz Adaptativa: Soporte para Modo Oscuro y Modo Claro.

3. ARQUITECTURA DEL CÓDIGO (LÓGICA)
--------------------------------------------------------------------------------
El proyecto sigue el principio de responsabilidad única y polimorfismo:

- PAQUETE 'core':
  * ImageFilter (Interfaz): Define el método 'apply(BufferedImage)'. Esto 
    permite que cualquier filtro nuevo que se cree sea compatible con la UI 
    automáticamente.

- PAQUETE 'filters':
  * Contiene los algoritmos. Cada filtro es una clase que implementa la 
    lógica de bits para transformar píxeles.
  * Ejemplos: GrayscaleFilter, RetroEffectFilter, ConvolutionFilter (matrices).

- PAQUETE 'ui':
  * MainFrame: Ventana principal para edición rápida y análisis comparativo.
  * AdvancedEditorFrame: Ventana secundaria con historial de efectos (Stack)
    y previsualizaciones dinámicas.

4. LÓGICA INTERNA DE PROCESAMIENTO
--------------------------------------------------------------------------------
Todos los filtros funcionan bajo la misma lógica técnica:

1. Descomposición ARGB: Se extraen los canales de cada píxel mediante 
   desplazamiento de bits.
   Ejemplo (Canal Rojo): (pixel >> 16) & 0xFF
2. Aplicación de Fórmulas:
   - Grises: Se usa la fórmula de luminosidad (0.21R + 0.71G + 0.07B).
   - Convolución: Se calcula un promedio ponderado de los píxeles vecinos 
     usando una matriz de 3x3.
   - Retro: Se aplica una reducción de bits para cuantizar los colores.
3. Recomposición: Se vuelven a unir los canales usando el operador OR (|).

5. LIBRERÍAS UTILIZADAS
--------------------------------------------------------------------------------
- Java Swing & AWT: Base fundamental para la interfaz gráfica y el manejo 
  de BufferedImages.
- FlatLaf (Flat Dark/Light): Librería moderna para el Look & Feel de la 
  aplicación, dándole ese aspecto premium estilo Lightroom.
- ImageIO: Para la lectura (Cargar) y escritura (Guardar) de archivos PNG/JPG.

6. MODO DE USO (PASO A PASO)
--------------------------------------------------------------------------------
1. Cargar Imagen: Usa el botón "Cargar" en la barra superior.
2. Filtros Rápidos: Usa el panel derecho para aplicar efectos de un solo clic.
3. Análisis: Selecciona una opción en "VISTAS DE ANÁLISIS" para ver cómo 
   varía un filtro según su nivel de intensidad.
4. Modo Apilado: Haz clic en "Modo Apilado (Avanzado)" para entrar al editor
   donde puedes mezclar filtros.
5. Guardar: Una vez estés satisfecho, usa el botón "Guardar" para exportar 
   tu resultado en formato PNG.

7. NOTAS PARA LA PRESENTACIÓN
--------------------------------------------------------------------------------
- Destaca que el diseño es "Responsivo" (los elementos se ajustan al tamaño).
- Menciona que el Historial del Editor Avanzado usa una estructura de datos
  tipo "Pila" (Stack) para permitir el "Deshacer".
- Explica que el procesamiento es eficiente porque trabaja directamente sobre
  el mapa de bits de la memoria RAM.
